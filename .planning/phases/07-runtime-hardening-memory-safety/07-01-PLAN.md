# Plan 07-01: Extend Schema and Contracts for Memory Safety

## Goal

Add the Phase 7 schema and contract surface for memory tiers, validation state, contamination/quarantine metadata, feasibility checks, and richer session-envelope summaries.

## Tasks

1. Extend SQLAlchemy enums/models for memory tier, validation status, contamination risk, quarantine reason, and feasibility-check persistence.
2. Extend vault and orchestration Pydantic contracts to expose the new fields and summaries.
3. Keep existing fields compatible so the current runtime can continue to boot while the new logic lands in later plans.

## Verification

- Model metadata imports cleanly.
- Session/vault serialization includes the new fields without breaking existing tests.
- No migration-breaking assumptions are introduced in the code paths used by current local verification.
