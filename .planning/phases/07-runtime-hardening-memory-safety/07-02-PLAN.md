# Plan 07-02: Implement Interruption-Aware Delta Replanning

## Goal

Teach the runtime to accept explicit interruption intents and reroute execution from the earliest affected stage instead of using one generic revision path.

## Tasks

1. Add interruption intent inputs and runtime transition helpers.
2. Persist replan summaries in the session envelope.
3. Route runtime transitions differently for JD changes, requirement changes, fact clarifications, risk flags, and draft-only wording edits.

## Verification

- Runtime tests show the correct `replanFromStage` for each interruption type.
- Existing happy-path flows still complete without invoking interruption paths.
- Interrupt traces are persisted and visible in session artifacts/events.
